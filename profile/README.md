## 一、先搞清楚：VPS 连不上不一定是 IP 被封

先别急着下结论。搬瓦工 VPS 连不上，大致有这几种可能：

**可能性一：IP 被封。** 国内访问不通，但国外能正常连。通常是因为 VPS 被用于某些不当用途，触发了防火墙屏蔽。

**可能性二：VPS 没有正常开机。** 国内国外都连不上。可能是 VPS 崩了或者意外关机。

**可能性三：SSH 端口被封。** IP 本身是好的，但 SSH 端口被单独屏蔽了。国内 TCP 不通，国外 TCP 通，ICMP（ping）两边都通——这种情况容易被误判为 IP 被封。

**可能性四：流量用完被暂停。** 搬瓤工会在月流量用尽后自动暂停 VPS，等到次月流量重置才能恢复。

所以，测试 IP 是否被封的第一步，是排除其他可能性。

## 二、搬瓦工测试 IP 是否被封：三种方法

### 方法一：国内 + 国外端口扫描对比法（经典通用）

这是搬瓦工官方客服推荐过的检测方式，思路很直接——从国内和国外各扫一次你的 SSH 端口，对比结果。

**第一步：国内扫描**

打开国内端口检测工具，比如站长之家端口扫描（[tool.chinaz.com/port/](http://tool.chinaz.com/port/)），输入你的 VPS IP 和 SSH 端口（默认 22，但搬瓦工分配的端口通常是随机五位数，在 KiwiVM 面板能查到）。

点击「开始扫描」，记下结果，称之为 R1。

**第二步：国外扫描**

打开国外端口检测工具，比如 [yougetsignal.com/tools/open-ports/](https://www.yougetsignal.com/tools/open-ports/)，同样输入 IP 和端口，点击 Check，记下结果，称之为 R2。

**结果判断如下：**

- R1 = 打开，R2 不管：你的 IP 完全正常，连不上是其他原因。
- R1 = 关闭，R2 = open：你的 IP 被封了，国内访问被屏蔽。
- R1 = 关闭，R2 = closed：VPS 可能没有正常开机，先检查 KiwiVM 面板里 VPS 的运行状态。

---

### 方法二：port.ping.pe（最直观、推荐）

打开 [port.ping.pe](http://port.ping.pe/)，在输入框里填写「IP:端口」，比如 `1.2.3.4:12345`，然后点击测试。

这个工具会从全球多个节点同时测试你的端口可达性，包含中国大陆节点和境外节点。测试结果会用颜色标出来：

- 境外节点显示 **successful（绿色）**，大陆节点显示 **failed（红色）**：IP 或端口被封了。先换个 SSH 端口试试，如果换端口还是大陆全挂，那就是 IP 本身被封了。
- 全部 **successful**：端口没问题，连不上是其他原因。
- 全部 **failed**：端口没有开放，可能是 VPS 宕机或者防火墙设置有问题。

注意：结果里如果有「pinger offline」，是该测试节点自身的问题，和你的 VPS 无关，直接忽略即可。

### 方法三：搬瓦工 KiwiVM 官方检测工具（最权威）

这是搬瓦工自己提供的检测方式，结果最为准确。

**步骤如下：**

1. 登录搬瓦工账号，进入「Services → My Services」，找到对应的 VPS，点击「KiwiVM Control Panel」按钮，进入 KiwiVM 面板。

2. 在 KiwiVM 面板已打开的状态下（必须先保持登录状态），打开这个 URL：


https://kiwivm.64clouds.com/main-exec.php?mode=blacklistcheck


（注意：如果直接访问这个链接会提示 Session timed out，必须先登录 KiwiVM 面板再访问。）

3. 点击「Test Main IP」按钮。

4. 结果只有两种：
   - **IP NOT BLOCKED**：IP 正常，未被封。
   - **IP BLOCKED**：IP 被封了。

这个工具检测的是 GFW（中国防火长城）黑名单，是目前最直接的官方判断依据。如果显示 NOT BLOCKED 但还是连不上，说明问题出在 VPS 本身，建议直接在 KiwiVM 里重启一下。
